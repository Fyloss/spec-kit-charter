# Code Quality

## Core Principles

- Write code that is clear, simple, and maintainable.
- Favor readability over clever or overly compact solutions.
- Keep functions and modules focused on a single responsibility.
- Eliminate duplication whenever practical.
- Use consistent naming and formatting throughout the project.
- Prefer composition and modular design over tightly coupled code.
- Handle errors explicitly and fail with clear, actionable messages.
- Validate inputs and avoid relying on implicit assumptions.
- Write deterministic code with minimal side effects.
- Optimize for correctness first, performance second; optimize only when justified.
- Remove dead code, unused dependencies, and obsolete comments.
- Document non-obvious decisions and public interfaces.
- Ensure new code is testable and does not reduce overall quality.
- Leave the codebase cleaner than you found it.

## Before Completing Any Change

- Verify the solution is correct.
- Check that the implementation is as simple as possible.
- Ensure consistency with the existing architecture and conventions.
- Consider edge cases and failure scenarios.
- Review the code as if it were a pull request before considering it complete.
